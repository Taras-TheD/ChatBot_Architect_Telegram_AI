# ChatBot_Architect_Telegram_AI

**ChatBot_Architect_Telegram_AI** — это экспериментальный проект личного Telegram-бота, который:
- общается от лица автора;
- использует продуманную архетипическую личность (Нуит / Хадит / Ра-Хоор);
- поддерживает режимы: общение, флирт, философия, мягкая терапия, саморефлексия;
- умеет опираться на AI (OpenAI) для генерации "оракульных" ответов и "пророчеств";
- в будущем — анализирует эмоциональное состояние собеседника, динамику диалогов и пишет еженедельные отчёты.

## Структура проекта

- `docs/` — концептуальные документы проекта:
  - описание цели бота;
  - поведенческая модель;
  - техническая архитектура;
  - диалоговая система;
  - план создания (Документ 11);
  - и другие вспомогательные файлы.

- `src/` — исходный код бота:
  - `main.py` — точка входа (запуск бота).
  - `bot/`, `core/`, `modules/`, `utils/` — модули, которые будут отвечать за:
    - интеграцию с Telegram,
    - работу с OpenAI,
    - память, аналитику, личность и логику общения.

- `infra/` — инфраструктура для деплоя (будет использоваться позже: Railway / Docker).
- `experiments/` — эксперименты, тестовые скрипты.
- `tests/` — модульные тесты (планируются).

## План развития (в общих чертах)

1. **MVP**:
   - Бот в Telegram, работающий через long polling.
   - Команды: `/start`, `/help`, `/oracle`.
   - Связка с OpenAI для генерации ответов.
2. **Личность и архетипы**:
   - реализация персонажа, основанного на документах из `docs/`.
3. **Память и аналитика**:
   - хранение истории диалогов,
   - анализ эмоционального состояния и динамики.
4. **Отчёты и "магическая" аналитика**:
   - еженедельные сводки, рекомендации, "оракульные" комментарии.
5. **Инфраструктура и деплой**:
   - перенос бота на удалённый сервер (Railway или аналог).
